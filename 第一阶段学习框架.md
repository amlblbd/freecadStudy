基于FreeCAD项目的复杂性，我建议你按以下详细计划进行第一阶段的学习：

## 📖 **第一阶段：基础架构理解（建议2-3周）**

### 🎯 **第1周：环境搭建与整体认知**

**1.1 环境准备**

```bash
# 确保你已经完成了以下准备
- 成功编译FreeCAD项目
- 熟悉基本的Git操作
- 配置好IDE（推荐VS Code或Visual Studio）
```

**1.2 架构概览学习**

- **阅读核心文档**：
  - `src/Doc/primary-groups.dox` - 理解CORE、WORKBENCHES、EMBEDDED分组
  - `src/App/core-app.dox` - 深入学习App层架构
  - `src/Base/core-base.dox` - 了解基础服务
  - `src/Gui/core-gui.dox` - 掌握GUI层设计

**1.3 模块结构探索**

```
重点研究以下目录结构：
src/
├── Base/     # 基础服务层
├── App/      # 应用逻辑层  
├── Gui/      # 图形界面层
└── Mod/      # 功能模块层
    ├── Part/        # 基础几何
    ├── PartDesign/  # 特征建模
    ├── Sketcher/    # 草图系统
    └── Draft/       # 2D绘图
```

### 🛠️ **第2周：核心概念深入**

**2.1 文档-对象系统**

- **关键类理解**：
  - `App::Document` - 文档管理器
  - `App::DocumentObject` - 所有对象的基类
  - `App::Property` - 属性系统基础
  - `App::PropertyContainer` - 属性容器

**2.2 属性系统学习**

- 阅读`src/App/Property.h`和`src/App/PropertyContainer.h`
- 理解静态属性与动态属性的区别
- 学习属性的持久化机制

**2.3 几何内核基础**

- 了解`Part::Feature`类层次结构
- 学习`TopoDS_Shape`与FreeCAD对象的映射关系
- 理解`execute()`方法的作用

### 🔍 **第3周：代码实践与跟踪**

**3.1 简单功能跟踪**
选择你已经了解的功能进行代码跟踪：

**跟踪"创建立方体"的完整流程**：

// 从GUI命令开始

**3.2 调试技巧实践**

- 在`Part::Box::execute()`方法中设置断点
- 观察属性值的传递过程

**3.3 Python接口探索**

```python
# 在FreeCAD Python控制台中实践
import FreeCAD as App
import Part

# 创建文档和对象
doc = App.newDocument()
box = doc.addObject("Part::Box", "MyBox")
box.Length = 10
box.Width = 20
box.Height = 30

# 观察属性变化
print(box.PropertiesList)
print(box.Shape)
```

### ⚠️ **学习重点提醒**

**务必掌握的核心概念**：

1. **文档是核心**：所有对象都存在于文档中
2. **属性驱动**：对象行为由属性值变化触发
3. **延迟计算**：execute()方法在需要时才被调用
4. **分层设计**：UI命令 → 应用逻辑 → 几何内核

**建议的学习方法**：

- **理论+实践结合**：阅读文档后立即用代码验证
- **从简单到复杂**：先理解立方体，再研究复杂特征
- **多问为什么**：每个设计决策都有其合理性
- **记录学习笔记**：建立自己的知识图谱
